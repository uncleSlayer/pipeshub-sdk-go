# PersistedSemanticSearch

Persisted search document as returned to clients.
`records` is a string-valued map: each value is `JSON.stringify(<record>)`,
keyed by the source record's `_id` or `_key`. Clients must `JSON.parse`
each value to recover the underlying record object (whose shape resembles
`SemanticSearchGraphRecord`). This intentionally differs from the
`searchResponse.records` array on the POST `/search` response, which is
passed through from the retrieval service untouched.
